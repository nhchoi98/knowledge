# 프로그래밍 언어

# 기초 용어

## 변수 (Variable)

하나의 값을 저장하기 위해 확보한 메모리 공간 자체를 의미한다. 값의 위치를 가리킨다. 

## 변수 선언 (Variable declare)

값을 저장하기 위한 메모리 공간을 확보 (allocate), 변수 이름과 확보된 메모리 공간의 주소를 연결 (Name binding)해서 값을 저장할 수 있게 준비하는 것이다. 

선언 단계에서는 변수의 존재를 알리고, 초기화 단계에서는 값을 저장하기 위한 메모리 공간을 확보한다. 

## 선언문 (Statement)

선언문은 **프로그래밍 언어나 문서에서 어떤 것을 “선언(declare)”하는 문장**

을 말해요. 즉, 컴파일러나 인터프리터에게 “이런 이름의 변수, 함수, 클래스, 상수 등이 있다”라고 알려주는 역할을 합니다.

### 예시 (JavaScript 기준)

- **변수 선언문**
    
    ```jsx
    let x;
    const y = 10;
    var z = "hello";
    ```
    
    → 여기서는 `x`, `y`, `z`라는 이름이 앞으로 사용될 수 있다고 선언한 것.
    
- **함수 선언문**
    
    ```jsx
    function greet() {
      console.log("Hello");
    }
    ```
    
    → `greet`라는 함수를 정의하고 사용 가능하게 만드는 선언문.
    
- **클래스 선언문**
    
    ```jsx
    class Person {
      constructor(name) {
        this.name = name;
      }
    }
    ```
    

## 리터럴(Literal)

***“그 자체로 값(value)을 나타내는 표기법”이에요.**

즉, 변수를 거치거나 계산하지 않고, 코드에 직접 적힌 값 그대로 의미를 가지는 것.

이 자체도 값이므로, 표현식으로 취급된다. 

### 객체 리터럴

- **중괄호 `{}` 안에 key–value 쌍을 직접 나열해서 객체를 만드는 방식**이에요.
- 즉, “객체를 직접 코드 안에 적어놓는 것”이 **객체 리터럴**.

→ JSON과 차이점은, Key에 네이밍 컨벤션에 따라서 ‘ ‘ 가 붙냐 안붙냐 차이가 있음 

→ JSON은 또한, 매서드(함수), undefined, Symbol 표현 불가 

## 표현식(Expression)

- 어떤 **값으로 평가(evaluate)** 될 수 있는 코드.
- 실행하면 반드시 결과(값)를 돌려줌.
- 문장(statement) 속에 포함될 수 있음.
- 표현식은 그 리터럴이나 변수, 연산 등을 조합해 “값을 산출하는 것”**이에요.

## 값 (Value)

- Expression이 Evaluate되어 생성된 결과

## 리터럴, 표현식, 값의 관계

```jsx
리터럴(Literal) ──┐
│
▼
Expression (표현식)
│   (evaluate)
▼
Value (값)
```

- **리터럴** = 값 자체를 코드에 직접 적은 것
- **표현식(Expression)** = 값으로 평가될 수 있는 모든 코드
- **값(Value)** = 표현식을 평가(Evaluate)한 결과
- **평가(Evaluate)** = 표현식을 실행해서 값을 얻는 과정

## 식별자  (Identifier)

프로그래밍 언어에서 변수, 함수, 클래스, 객체 등과 같이 "무언가를 구분하기 위해 붙이는 이름

- 숫자로 시작할 수 없음 (`1name` ❌).
- 예약어(reserved word)는 사용할 수 없음 (`if`, `for`, `class` ❌).

## 문맥(Context)

코드가 실행되는 환경을 뜻해요.

엔진이 코드를 실행할 때, 필요한 정보(변수, 함수 선언, this 값 등)를 담아두는 일종의 “박스”예요.

## 함수(Function)

- **하나의 이름 아래 묶어둔 실행 가능한 코드 블록**.
- 입력(**매개변수**)을 받아서, 내부에서 어떤 일을 수행하고, 결과(**반환값**)를 돌려줄 수도 있어요.
- *“재사용 가능한 로직 단위”**라고 이해하면 쉽습니다.

### 🔹 함수의 구성 요소

1. **이름** (선택적, 익명 함수일 수도 있음)
2. **매개변수(Parameter)**: 외부에서 함수에 전달하는 값
3. **본문(Body)**: 실행할 코드 블록
4. **반환값(Return value)**: 함수 실행 후 돌려주는 값 (없을 수도 있음)

### 고차함수 (Higher-Order Function, HOF)

**다른 함수를 인자로 받거나, 함수를 반환하는 함수**를 말해요.

즉, **함수를 다루는 함수**라고 이해하면 됩니다.

### 익명함수 (Anonymous function)

***컴퓨터 프로그래밍에서 식별자에 바인딩되지 않은 함수*** 정의이다. 

| 구분 | 문법 | 이름 유무 | 호이스팅 | 특징 |
| --- | --- | --- | --- | --- |
| **익명 함수** | `function() {}` | ❌ 없음 | X (표현식이면 불가) | 이름 없이 일회성 사용 |
| **화살표 함수** | `(a, b) => a + b` | 보통 ❌ | X | 짧고 간결, `this`를 상위 스코프에 바인딩 |
| **선언식 함수** | `function add() {}` | ✅ 있음 | O | 코드 어디서든 호출 가능 |
| **함수 표현식** | `const fn = function() {}` | 보통 ❌ | X | 변수에 담는 방식, 더 유연함 |

## 인터프리터 언어 (Interpreter Language)

- **프로그램을 한 줄씩 읽고 바로 실행하는 방식**을 사용하는 언어.
- 실행 전에 기계어(바이너리)로 전체를 “컴파일”하지 않고, 소스 코드를 인터프리터(해석기)가 **실시간으로 해석해서 실행**함

## ✅ 특징

1. **즉시 실행**
    - 코드를 작성하고 곧바로 실행 가능 (별도의 컴파일 단계 불필요).
    - 빠르게 테스트/디버깅 가능.
2. **이식성 좋음**
    - 인터프리터만 있으면 OS/환경에 관계없이 실행 가능.
3. **실행 속도**
    - 컴파일 언어(C, C++)보다 보통 느림.
    - 이유: 실행할 때마다 코드를 해석해야 하기 때문.
    - 하지만 요즘은 JIT(Just-In-Time) 컴파일러 같은 기술로 속도 개선됨 (예: V8 엔진).

## 📌 인터프리터 언어의 특징과 장점

### 1. **동적 타이핑 (Dynamic Typing)**

- 변수의 타입을 미리 정하지 않고 실행 시점에 결정됨.
- 예: JavaScript, Python 모두 변수 선언 시 타입을 안 적어도 됨.

### 3. **REPL 지원 (즉시 실행 환경)**

- Python의 **REPL**(`python` 명령어)이나 JS의 **브라우저 콘솔**, **Node.js** 환경처럼,
    
    코드를 한 줄씩 입력하고 곧바로 실행 결과 확인 가능.
    
- 실험, 테스트, 디버깅에 매우 유리.

### 4. **플랫폼 독립성**

- 인터프리터만 있으면 어디서든 실행 가능.
- Python은 **CPython 인터프리터**, JS는 **V8(크롬)**, **SpiderMonkey(파이어폭스)** 같은 엔진이 대신 실행해줌.
    
    👉 덕분에 "한 번 작성하면 어디서든 실행"이 쉬워짐.
    

### 5. **동적 기능 지원**

- 실행 도중에도 코드 구조를 바꾸거나 새로운 걸 추가할 수 있음.
    
    예: 객체에 실행 중에 속성 추가 가능 (JS/Python 모두 지원).
    

## 동적 타입 언어

- **실행 중(Run-time)에 변수의 타입이 결정**돼요.
- 변수를 선언할 때 타입을 명시하지 않고, 대입하는 값에 따라 타입이 달라져요.
- 코드가 간결하고 빠르게 작성 가능하지만, **실행 도중에 타입 오류**가 날 수 있어요.

## 정적 타입 언어

- **컴파일 시점(Compile-time)에 변수의 타입이 결정**돼요.
- 변수를 선언할 때 타입을 반드시 지정해야 하고, 이후 값이 그 타입과 맞지 않으면 오류가 발생해요.
- 실행 전에 타입 오류를 잡아줄 수 있어 안정성이 높아요.

# 객체지향 관련 용어

## 객체 (Class)

### 일급객체 (First class Object)

프로그래밍 언어에서 “값처럼 자유롭게 다룰 수 있는 대상”**을 말해요.

**🔹 일급 객체의 조건**

어떤 대상(예: 함수, 클래스, 숫자 등)이 아래 조건들을 만족하면 **일급 객체**라고 불러요.

- 변수나 데이터 구조에 담을 수 있다.
- 함수의 매개변수로 전달할 수 있다.
- 함수의 반환값으로 사용할 수 있다.

### 일급 함수 (First class function)

- 값 처럼 변수에 담고, 전달하고, 반환할 수 있는 함수

## 인스턴스 (Instance)

## 함수형 프로그래밍

하나의 프로그래밍 패러다임. 순수함수와 보조함수의 조합을 통해, 외부 상태를 변경하는 부수 효과를 최소화해서 불변성(Immutability)를 지향하는 프로그래밍 패러다임이다. 
프로그램을 “명령어의 나열”이 아니라, **데이터 변환의 흐름**으로 바라봄.

# 타입 관련

## Explict coersion(명시적 타업변환)

- 개발자가 **의도를 가지고 직접** 변환하는 것.
- 즉, **“나 지금 이 값을 바꿀 거야”**라고 코드에 드러나는 경우.
- 다른 말로 **Type Casting (타입 캐스팅)** 이라고 부르기도 해요.

## Implicit coersion (암묵적 타입 변환)

- 언어가 **자동으로 타입을 변환**하는 것.
- 개발자가 직접 변환을 쓰지 않아도, 문맥(Context)에 따라 언어가 알아서 바꿔줌.
- 종종 **예상치 못한 동작** 때문에 버그의 원인이 되기도 함.

## Type coersion

- **한 타입의 값을 다른 타입으로 바꾸는 모든 행위**를 말해요.
- 명시적 변환(Explicit)과 암묵적 변환(Implicit)을 모두 포함하는 **상위 개념**.
