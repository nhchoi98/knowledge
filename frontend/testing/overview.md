# FE 테스팅

### UI/UX 및 호환성 테스트 (깨짐 방지)

가장 기본적이지만, 가장 많이 놓치는 부분입니다.

- **크로스 브라우징 (Cross-Browsing):**
    - Chrome 외에 **Safari(특히 iOS/Mac)**, Firefox, Edge에서 테스트했나요?
    - *Note:* Safari는 날짜 포맷(`new Date`)이나 CSS Flexbox/Grid 처리 방식이 미묘하게 다를 때가 많습니다.
- **반응형 및 뷰포트(Viewport) 변화:**
    - 모바일, 태블릿, 데스크탑 해상도 확인.
    - **창 크기를 마우스로 빠르게 늘렸다 줄였다 할 때** 레이아웃이 깨지지 않나요?
    - 가로 모드(Landscape)로 전환했을 때 UI가 정상적인가요?
- **브라우저 줌(Zoom) 테스트:**
    - 화면을 **125%, 150%, 200%로 확대**했을 때 글자가 겹치거나 버튼이 사라지지 않나요?
- **시스템 테마 (Dark/Light Mode):**
    - OS 설정이 다크 모드일 때, 강제로 흰색 배경에 흰색 글씨가 나오거나 하진 않나요?

### 2. 네트워크 및 비동기 예외 처리 (Unhappy Path)

개발 환경은 네트워크가 빠르지만, 실제 환경은 그렇지 않습니다.

- **네트워크 스로틀링 (Throttling):**
    - 개발자 도구에서 'Slow 3G'로 설정 후 로딩 시 **스켈레톤 UI나 로딩 스피너**가 적절히 뜨나요?
    - 데이터가 늦게 올 때 버튼을 클릭하면 오류가 나나요?
- **오프라인 모드:**
    - 인터넷 연결을 끊고 액션을 취했을 때, "인터넷 연결을 확인해주세요" 같은 적절한 피드백(Toast/Modal)이 뜨나요? 아니면 앱이 멈추나요?
- **API 에러 (400, 500번대):**
    - 서버가 죽었거나(500), 요청이 잘못되었을 때(400) 무한 로딩에 걸리지 않고 에러 화면 처리가 되나요?
- **중복 요청 방지 (Debounce/Throttle):**
    - 저장/결제 버튼을 **빠르게 다닥다닥(Double Click)** 눌렀을 때, 요청이 여러 번 날아가나요? (매우 중요)

### 3. 입력 폼(Form) 및 데이터 조작

사용자는 개발자가 의도하지 않은 데이터를 입력하곤 합니다.

- **특수문자 및 이모지 입력:**
    - 이름이나 내용에 이모지(😊)나 SQL Injection 의심 문자(`'`, `;`)를 넣었을 때 정상 저장/출력 되나요?
- **초장문 텍스트 (Long Text):**
    - 입력 필드 제한보다 긴 텍스트를 복사/붙여넣기 했을 때 잘리거나 경고가 뜨나요?
    - 긴 텍스트가 화면에 렌더링될 때 UI 밖으로 튀어나가지 않고 말줄임(...) 처리가 되나요?
- **공백(White Space) 처리:**
    - 스페이스바만 입력해서 제출했을 때 통과되나요? (Trim 처리 확인)
- **자동 완성(Autofill) 배경색:**
    - 브라우저가 아이디/비번을 자동 완성했을 때, 인풋 창 배경색(주로 노란색/파란색)이 디자인을 해치지 않나요?

### 4. 상태 유지 및 내비게이션

SPA(Single Page Application)에서 자주 발생하는 문제입니다.

- **뒤로 가기/앞으로 가기:**
    - 필터링을 건 상태에서 상세 페이지로 갔다가, **'뒤로 가기'를 눌렀을 때 필터링 조건이 유지**되나요?
    - 모달(Modal)이 뜬 상태에서 '뒤로 가기'를 누르면 모달만 닫히나요, 아니면 페이지가 이동하나요?
- **새로고침 (F5):**
    - 중요한 작업(결제, 작성) 중간에 새로고침 했을 때 데이터가 다 날아가나요? (경고창 띄우기 필요 여부 확인)
    - URL에 쿼리 파라미터가 있는 상태에서 새로고침해도 해당 페이지가 잘 보이나요?
- **토큰 만료 (로그인 세션):**
    - 로그인 상태로 화면을 켜두고 한참 뒤(토큰 만료 후) 버튼을 눌렀을 때, 조용히 실패하나요 아니면 로그인 창으로 튕겨주나요?

### 5. 접근성 (Accessibility)

필수는 아니라고 생각할 수 있지만, 품질을 결정짓는 요소입니다.

- **키보드 내비게이션 (Tab 키):**
    - 마우스 없이 `Tab` 키만으로 모든 입력창과 버튼을 순서대로 이동하고 실행(Enter)할 수 있나요?
    - 포커스(Focus)가 어디에 있는지 시각적 테두리가 보이나요?
- **스크린 리더:**
    - 이미지에 `alt` 태그가 빠져있지는 않나요?
